# スキーマ療法カード・ワークスペース 設計書

日付: 2026-07-18
ステータス: 承認済み

## 目的

スキーマ療法で用いられるカードワーク(机にカードを広げ、自分に当てはまるものを選び、並べて内省する)をブラウザ上で再現する、個人のセルフヘルプ用ツール。

## 利用者と前提

- 利用者は本人のみ(セルフヘルプ用)。アカウント・サーバー・ネットワーク接続は不要。
- データはすべて利用者のブラウザ内(localStorage)に保存され、外部に送信されない。
- 対応環境: PC の Chrome / Edge を主対象とし、タッチ操作(タブレット)にも対応する。

## カードデッキ

ヤングの 18 の早期不適応スキーマを 1 枚ずつカードにした固定デッキ。各カードは以下を持つ。

- スキーマ名(日本語)
- 説明文(2〜3 文、平易な日本語)
- 所属領域(5 領域)と領域ごとのテーマカラー

| 領域 | スキーマ |
|---|---|
| I. 断絶と拒絶 | 見捨てられ/不安定、不信/虐待、情緒的剥奪、欠陥/恥、社会的孤立/疎外 |
| II. 自律性と行動の損傷 | 依存/無能、危険や病気に対する脆弱性、巻き込まれ/未発達の自己、失敗 |
| III. 制約の欠如 | 権利要求/尊大、自制と自律の欠如 |
| IV. 他者への追従 | 服従、自己犠牲、評価と承認の希求 |
| V. 過剰警戒と抑制 | 否定/悲観、感情抑制、厳密な基準/過度の批判、罰 |

カードデータは HTML 内に JavaScript 定数として同梱する。

## 画面フローと機能

3 画面を単一ページ内で切り替える(SPA、画面 = 状態)。

### 1. ホーム / セッション一覧

- 過去のワーク(セッション)が更新日時の新しい順にカード状に並ぶ。
- 各セッション: タイトル(既定は日付)、作成/更新日時、選択カード枚数、「開く」「削除」。
- 「新しいワークを始める」ボタンで新規セッションを作成し、カード選択画面へ。
- セッションが 1 件もない初回はこの画面を飛ばしてカード選択画面から始める。
- 削除は確認ダイアログを挟む。

### 2. カード選択画面

- 18 枚全カードをグリッド表示(領域ごとに色分け、領域見出し付き)。
- カードクリックで選択トグル。選択中はチェックマーク+強調枠。
- 画面下部に固定バー:「◯枚選択中」表示と「確定」ボタン(0 枚時は無効)。
- 既存セッションの選び直しで戻ってきた場合は現在の選択状態を反映して表示する。

### 3. ワークキャンバス画面

確定後、選択したカードのみが「机」の上に表示され、自由に並べてワークを行う。

- **初期配置**: 新規確定時はカードを机中央に扇状/グリッド状に整列配置。再開時は保存済みの位置を復元。
- **ドラッグ移動**: Pointer Events でカードを自由な位置へ移動。ドラッグ中はカードがわずかに浮き上がり(影+拡大)、最前面(z 順の先頭)に移動する。
- **ズーム**: マウスホイール(カーソル位置基準)/ ピンチ。ツールバーに +/− ボタンも置く。範囲 0.3〜3.0 倍。
- **チルト**: ツールバーのスライダーで机全体を 0°(真上)〜60°(斜め俯瞰)に傾ける。CSS の `perspective` + `rotateX` による 3D 俯瞰。
- **パン**: 机の余白をドラッグして視点を移動。
- **視点リセット**: ズーム 1.0 / チルト 0° / パン中央に戻すボタン。
- **カードを選び直す**: 選択画面(2)に戻る。すでに配置済みのカードの位置は保持し、追加されたカードだけ初期配置、外されたカードは机から取り除く。
- **ホームへ戻る**: 自動保存済みなのでそのまま一覧へ。

### 選び直し時の位置保持ルール(明確化)

- 選択継続カード → 座標・z 順を維持
- 新規追加カード → 机中央付近の空き位置に整列配置
- 選択解除カード → placements から削除

## データモデルと保存

localStorage キー: `schemaTherapyCard.v1`

```json
{
  "version": 1,
  "sessions": [
    {
      "id": "ULID風の一意文字列",
      "title": "2026-07-18 のワーク",
      "createdAt": "ISO8601",
      "updatedAt": "ISO8601",
      "selectedCardIds": ["abandonment", "defectiveness"],
      "placements": {
        "abandonment": { "x": 120, "y": 80, "z": 2 }
      },
      "view": { "zoom": 1.0, "tilt": 30, "panX": 0, "panY": 0 }
    }
  ]
}
```

- 座標 (x, y) は机平面上の論理座標(ズーム・チルト非依存)。z は重なり順の連番。
- 保存タイミング: 選択確定・カード移動終了(pointerup)・視点変更・タイトル変更のたびに 500ms デバウンスで全体を書き戻す。
- カードの本文はデッキ定数から引くため保存しない(ID のみ保存)。

## 技術構成

- **`index.html` 1 ファイル**に CSS / JavaScript / カードデータをすべて内包。依存ライブラリゼロ、ビルド不要。ファイルをダブルクリック(`file://`)で動作する。
- 画面切り替えは状態変数 + セクション表示切替(ルーターなし)。
- 3D 表現: ビューポート要素に `perspective`、その子の「机」要素に `rotateX(tilt) scale(zoom) translate(panX, panY)` を適用。カードは机要素の絶対配置の子。
- **座標逆変換**: ドラッグ時はスクリーン座標の移動量を机平面の論理座標へ逆変換する(x はズームで除算、y はズームと `cos(tilt)` で除算する近似。perspective による遠近誤差は、掴んだ瞬間のポインタとカードの相対位置を固定する方式で吸収する)。
- タッチ: Pointer Events で統一的に扱い、2 本指ピンチ/パンに対応。

## エラー処理

- localStorage が使えない(プライベートモード等) → 起動時に警告バナーを表示し、メモリ上のみで動作(リロードで消える旨を明示)。
- 保存データの JSON 解析失敗 → 壊れたデータを `schemaTherapyCard.v1.backup` に退避してから初期化し、その旨を通知。
- 容量超過(QuotaExceededError) → 保存失敗を警告表示(18 枚+座標のみなので実際にはほぼ起きない)。

## 検証方針

自動テスト基盤は持たない(単一 HTML・依存ゼロのため)。実装後にブラウザで以下の一連の流れを実際に操作して確認する。

1. 初回起動 → 18 枚全カードが領域別に表示される
2. 複数枚選択 → 確定 → 選択カードのみが机に整列表示される
3. ドラッグで自由配置、ズーム、チルト、パン、視点リセットが機能する
4. リロード → ホームに前回セッションが表示され、開くと配置・視点が復元される
5. カードを選び直す → 既存配置の保持/追加/削除ルールどおりに動く
6. セッション削除、新規作成が機能する
