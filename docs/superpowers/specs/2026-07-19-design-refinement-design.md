# スキーマ療法カード デザイン刷新 設計書(案A+B折衷)

日付: 2026-07-19
対象: `index.html`(単一ファイル構成のまま)
方針: 「工芸品カードデッキ」(案A)を核に、「エディトリアル・ミニマル」(案B)のタイポ整備とダークモードを取り込む。

## 目的

現行の「落ち着いたセラピー風」デザインを、実物のセラピーカードを机に広げている感覚が伝わる本格的な意匠に洗練させる。機能・操作・データ形式は変更しない。

## 変更しないもの(制約)

- 機能と操作フロー: ホーム → カード選択 → 机(ドラッグ・ズーム・チルト・パン)。
- localStorage のキーとデータ形式(`schemaTherapyCard.v1`)。
- 単一 `index.html`、ビルド不要、`file://` で動作。
- 画像ファイルは追加しない(テクスチャ・紋様はすべてインライン SVG / CSS で生成)。
- `prefers-reduced-motion: reduce` でアニメーションを無効化する既存方針。

## 1. デザイントークン

### 配色

- 基調を「生成り(紙)・墨(文字)・金茶(装飾)」の3系統に整理する。
  - 金茶(`#a8845c` 系)を新設し、カードの内枠罫線・ローマ数字・飾り罫に使う。
  - 既存のセージ色はボタン等の操作系アクセントとして残す。
  - 領域5色は彩度を一段落として紙になじませる(色相は現行を踏襲)。
- すべての色を CSS 変数として定義し、ライト/ダークで切り替える。

### ダークモード

- `@media (prefers-color-scheme: dark)` による自動切替(手動トグルは設けない)。
- ダーク時: 机は暗い胡桃材、カードは暖色がかった紙色を保つ(「夜の書斎で紙のカードを見る」表現)。UI 面(ホーム・選択画面)は暗い生成り地に明るい墨色文字。

### タイポグラフィ

- 見出し用に Shippori Mincho を Google Fonts から読み込む(`<link>`、`display=swap`)。オフライン・`file://` 時はシステム明朝(游明朝/ヒラギノ明朝)へ自動フォールバック。JS での検出は行わず、font-family のフォールバック連鎖のみで実現する。
- 5段のタイポスケールを定義: h1 / 領域見出し / カード名 / 本文 / 注釈。字間(見出し `.06em` 程度)と余白リズムを再設計する。

## 2. カードの意匠(選択画面・机で共通)

- 縦長比 5:7。机上は約 150×210px、選択画面のグリッドも縦長カードにする。
- カードの構造(上から):
  1. 上部中央にローマ数字の通し番号(Ⅰ〜ⅩⅧ、金茶)と領域紋様。
  2. カード名(明朝・中央寄せ)。
  3. 罫線(短い金茶のセパレータ)。
  4. 説明文(小さめの本文)。
- 外枠の内側に金茶(または領域色)の内枠罫線を 1px で引き、二重枠にする。
- 領域紋様: 領域ごとに異なる小さなインライン SVG シンボル。
  - 断絶と拒絶 = 途切れた環
  - 自律性と行動の損傷 = 芽
  - 制約の欠如 = 天秤
  - 他者への追従 = 重なる二つの円
  - 過剰警戒と抑制 = 山形
- 紙テクスチャ: SVG `feTurbulence` を data URI 化して background-image に低不透明度で重ねる。
- カード裏面のデザインは作らない(YAGNI)。
- 選択状態の表現(チェックマーク・枠強調)は現行の仕組みを踏襲し、意匠のみ新デザインに合わせる。

## 3. 机の質感と光

- 木目: CSS グラデーションと SVG ノイズの合成で生成。ライト = 明るいオーク、ダーク = 胡桃。
- 環境光: 上方からの光だまり(radial-gradient)と周辺減光を現行より強める。
- 落ち影のチルト追従: ステージのチルト角に応じて、影のYオフセット・ぼかし量を表す CSS 変数(1個)を JS から更新し、全カードの box-shadow に反映する。カードごとの個別計算は行わない。

## 4. モーション

すべて `prefers-reduced-motion: reduce` で無効化する。

- 画面遷移: `#app` の短いフェードイン(120ms 程度)。
- 選択画面: カードが順に浮かび上がる stagger 表示(1枚あたり 20〜40ms ずらし)。
- 机: 確定後、カードが順に「置かれる」(scale 1.06→1 + フェード、stagger)。初回描画時のみで、ドラッグ等の再描画では発火させない。
- ドラッグ中: わずかな浮き(scale 1.05)+ 微小回転 + 影拡大。

## 5. 各画面の磨き込み

- ホーム: h1 の下に金茶の飾り罫(細線+中央の小さなダイヤ)。セッション一覧の各項目に、選択済みカードの領域色チップを枚数分表示する(最大8個、超える分は「+n」で示す)。
- 選択画面: 領域見出しを「紋様 SVG + 明朝見出し + 罫線」の章扉風にする。
- ツールバー: 構成・操作は不変。ダークモード対応と配色調整のみ。

## 6. 実装・検証方針

- 変更は原則 CSS と少量の HTML 生成部。JS ロジックの変更は (a) チルト影の CSS 変数更新、(b) 初回描画アニメーション用クラス付与、の2点のみ。
- 検証: `node --check` による構文確認、headless Chrome(`--screenshot`)でライト/ダーク × ホーム/選択/机(チルトあり)のスクリーンショット確認。
- 完了後は既存フローどおり main へ ff-merge し、GitHub Pages を更新する。
